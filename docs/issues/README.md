## When to Add Issues

Add an issue document when:
- You've encountered and solved a non-trivial bug
- The solution wasn't obvious and took investigation
- Future developers might encounter the same issue
- The fix involved understanding library/framework quirks

## Issue Template

```markdown
# Feature - Issues and Solutions

Overview of what was being implemented.

---

## Issue N: Brief Title

### Problem
Error message or description of what went wrong.

### Solution
Step-by-step fix with code examples.

**Files:** List of affected files

---
```
